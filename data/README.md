# Dataset Instructions

This folder is reserved for the datasets used by this project.

The actual dataset files are **not committed to GitHub**. To run the notebooks locally, download or place the required files under this folder using the structure shown below.

This project was originally developed for a Big Data Mining / High-Scale Analytics assignment with three tasks:

1. news article text classification,
2. nearest-neighbor search with Jaccard similarity, MinHash, and Locality Sensitive Hashing (LSH),
3. Dynamic Time Warping (DTW) for time-series similarity.

## Dataset Sources

### Parts 1 and 2: Text Classification / LSH Dataset

Parts 1 and 2 use the **BigData2024 classification** Kaggle competition dataset:

```text
https://www.kaggle.com/competitions/bigdata2024classification/data
```

The assignment PDF describes this dataset as a news-article classification dataset with a training file containing `Id`, `Title`, `Content`, and `Label`, and a test file without the `Label` field. The public Kaggle competition page currently provides the files as:

```text
train.csv
test_without_labels.csv
```

A Kaggle account and competition access may be required before downloading the files.

### Part 3: DTW Time-Series Dataset

Part 3 uses a separate time-series test file for Dynamic Time Warping. The assignment PDF states that a test dataset is provided and that each row contains two time series, `seq_a` and `seq_b`. However, the source line in the PDF is listed as:

```text
The data is available at: TBA
```

Therefore, this repository does **not** claim a verified public download link for the DTW dataset. To reproduce the DTW section, place the assignment-provided file locally as shown in the expected structure below.

## Expected Local Structure

After obtaining the required files, place them locally using the following structure:

```text
data/
├── bigdata2024classification/
│   ├── train.csv
│   └── test_without_labels.csv
│
├── timeseriesexercise3/
│   └── dtw_test.csv
│
└── pickle-files/
    ├── processed_data.pkl
    └── processed_test_data.pkl
```

## Notes on Each Folder

### `bigdata2024classification/`

This folder should contain the Kaggle text-classification files used in Parts 1 and 2:

```text
train.csv
test_without_labels.csv
```

These files are used for:

- text classification with Bag-of-Words features,
- SVM and Random Forest models,
- K-NN with Jaccard similarity,
- MinHash / LSH nearest-neighbor search.

### `timeseriesexercise3/`

This folder should contain the DTW input file:

```text
dtw_test.csv
```

The expected file contains rows with two time-series columns, commonly referred to as:

```text
seq_a
seq_b
```

The notebook computes DTW distances between these two sequences and writes the required output file.

### `pickle-files/`

This folder is optional. It is used for locally generated preprocessing caches, such as:

```text
processed_data.pkl
processed_test_data.pkl
```

These files are not original dataset files. They can be regenerated by running the preprocessing cells in the notebook, so they should not be committed to GitHub.

## Path Notes

The original notebook was partly developed in Kaggle-style and local Linux environments, so some cells may contain paths such as:

```text
/kaggle/input/bigdata2024classification/train.csv
/kaggle/input/bigdata2024classification/test_without_labels.csv
/kaggle/input/pickle-files/processed_data.pkl
/kaggle/input/timeseriesexercise3/dtw_test.csv
/home/.../exercise1_dataset/test_without_labels.csv
```

When running the project locally from this repository, update those paths to the local `data/` structure above, for example:

```python
pd.read_csv("../data/bigdata2024classification/train.csv")
pd.read_csv("../data/bigdata2024classification/test_without_labels.csv")
pd.read_csv("../data/timeseriesexercise3/dtw_test.csv")
```

If the notebook is executed from the repository root instead of from inside the `notebooks/` folder, use paths such as:

```python
pd.read_csv("data/bigdata2024classification/train.csv")
```

## GitHub Policy

The dataset files are intentionally excluded from this repository because they are large and/or externally provided.

Only this `README.md` file should be committed inside the `data/` folder. The actual CSV and pickle files should remain local.
